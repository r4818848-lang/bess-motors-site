## 2. Вход в CRM (продакшен)

### Шаг 2.1. Откройте браузер

Рекомендуется **Google Chrome** на компьютере (не телефон для первого входа).

### Шаг 2.2. Очистите старый вход (если уже пробовали)

1. Откройте **новое окно инкогнито**: `Ctrl + Shift + N`.
2. Или: Настройки Chrome → Конфиденциальность → Очистить данные → только для `bess-motors.com`.

### Шаг 2.3. Откройте страницу входа для CRM

В адресной строке:

```
https://www.bess-motors.com/cabinet?crm=1
```

Должна появиться жёлтая подсказка: **«Во втором поле — пароль администратора»**.

### Шаг 2.4. Введите данные с Vercel

| Поле на экране | Что вводить |
|----------------|-------------|
| **Телефон** | Значение `ADMIN_PHONE` из Vercel (например `+48888838688` или `+48 888 838 688`) |
| **Второе поле** | Значение `ADMIN_PASSWORD` из Vercel — это **пароль**, не госномер авто |

### Шаг 2.5. Нажмите «Войти»

- Успех → откроется **CRM**: https://www.bess-motors.com/crm  
- Ошибка → см. [раздел 8](#8-если-не-входит--криво-работает)

### Шаг 2.6. Не путайте с клиентом

| Кто | Телефон | Второе поле |
|-----|---------|-------------|
| **Администратор CRM** | ADMIN_PHONE | ADMIN_PASSWORD |
| **Клиент (кабинет)** | Телефон клиента | Госномер авто |

## 3. Переменные на Vercel

### Откуда брать каждое значение (шпаргалка)

| Переменная в Vercel | Откуда взять | Куда вставить |
|---------------------|--------------|---------------|
| **`ADMIN_PHONE`** | **Вы сами придумываете** — любой ваш рабочий телефон в формате Польши. Это логин в CRM, не телефон с вывески (можно тот же). Пример: `+48888838688` или `+48791257229`. | Vercel → Settings → Environment Variables → имя `ADMIN_PHONE` |
| **`ADMIN_PASSWORD`** | **Вы сами придумываете** — пароль только для входа в CRM (латиница + цифры, 8+ символов). Нигде в интернете его нет, пока вы не создадите. Запишите в надёжное место. | Vercel → имя `ADMIN_PASSWORD` |
| **`JWT_SECRET`** | **Вы сами генерируете** — случайная длинная строка. Не из Supabase и не из Meta. См. команду PowerShell ниже или любой генератор паролей (32+ символа). | Vercel → имя `JWT_SECRET` |
| **`SUPABASE_URL`** | Сайт **Supabase** → ваш проект → **Project Settings** (шестерёнка) → **Data API** → поле **Project URL**. Вид: `https://abcdefgh.supabase.co` | Vercel → имя `SUPABASE_URL` |
| **`SUPABASE_SERVICE_ROLE_KEY`** | Supabase → **Project Settings** → **API Keys** → ключ **service_role** / **Secret** (подпись `sb_secret_...` или длинный `eyJ...`). Копировать **Secret**, не publishable. | Vercel → имя `SUPABASE_SERVICE_ROLE_KEY` |
| **`NEXT_PUBLIC_SITE_URL`** | Просто адрес сайта: **`https://www.bess-motors.com`** (как в браузере, без слэша в конце). | Vercel → имя `NEXT_PUBLIC_SITE_URL` |

#### Подробно по каждому полю

**1. `ADMIN_PHONE` и `ADMIN_PASSWORD` — только вы**

- Это **не** берётся из Supabase, Vercel или Meta.
- Вы один раз решаете: «в CRM буду входить с этим телефоном и этим паролем».
- Телефон на сайте для клиентов (`+48 791 257 229`) может быть **другим** — для CRM можно свой номер.
- После сохранения в Vercel **именно эти** данные вводите на странице `/cabinet?crm=1`.

**2. `JWT_SECRET` — секрет подписи сессий**

- Нужен, чтобы после входа CRM вас не выкидывало.
- **Где взять:** сгенерировать на ПК (PowerShell):

```powershell
[Convert]::ToBase64String((1..48 | ForEach-Object { Get-Random -Maximum 256 }) -as [byte[]])
```

- Скопировать **всю** строку в Vercel. Никому не показывать.

**3. `SUPABASE_URL` и `SUPABASE_SERVICE_ROLE_KEY` — база CRM**

1. Откройте https://supabase.com и войдите.
2. Выберите проект BESS MOTORS (тот, куда загружались заказы).
3. Слева внизу **Project Settings** (иконка шестерёнки).
4. Вкладка **API** (или **Data API**):
   - **Project URL** → это `SUPABASE_URL`
   - **Project API keys** → **service_role** / **Secret key** → **Reveal** → скопировать → это `SUPABASE_SERVICE_ROLE_KEY`

![логика: не путать ключи]

| Ключ в Supabase | Использовать? |
|-----------------|---------------|
| **anon / publishable** | Нет (для сайта в браузере, не для CRM на сервере) |
| **service_role / secret** | **Да** → `SUPABASE_SERVICE_ROLE_KEY` |

**4. `NEXT_PUBLIC_SITE_URL`**

- Не «ищется» — вы **вписываете вручную**: `https://www.bess-motors.com`
- Должен совпадать с доменом в Vercel → **Domains**.

#### Что НЕ подставлять в эти поля

| Неправильно | Почему |
|-------------|--------|
| Госномер авто клиента | Это для кабинета клиента, не для CRM |
| Токен Telegram | Это `TELEGRAM_BOT_TOKEN`, другое поле |
| Пароль от Vercel / GitHub | Это вход в Vercel, не в CRM |
| Пароль от Supabase (логин в supabase.com) | Это вход в панель Supabase, не ключ API |

2. Скопируйте значение в **Блокнот** — посмотрите, нет ли пробела в конце.
3. При сомнении — задайте **новый пароль** (только латиница и цифры), сохраните, **Redeploy**, войдите с новым.

## 4. Выложить исправления на сайт (deploy)

Исправление входа в CRM должно быть **на сервере**. Если вы только меняли файлы на ПК — сайт в интернете старый.

### Вариант A — через GitHub (если проект уже подключён к Vercel)

**Шаг 4.1.** Откройте PowerShell:

```powershell
cd c:\Users\Asus\Desktop\website\bess-motors
git status
```

**Шаг 4.2.** Если Git ещё не настроен (один раз):

```powershell
git init
git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПОЗИТОРИЙ.git
```

**Шаг 4.3.** Сохранить и отправить:

```powershell
git add .
git commit -m "Fix CRM login and admin session"
git push origin main
```

**Шаг 4.4.** Vercel сам начнёт сборку (1–3 минуты).  
В Vercel → **Deployments** → дождитесь зелёной галочки **Ready**.

### Вариант B — Redeploy без кода

Если код уже на GitHub, но меняли только переменные Vercel:

1. Vercel → **Deployments**.
2. Последний успешный деплой → **⋯** → **Redeploy**.

### Вариант C — загрузка через Vercel вручную

Если Git нет: Vercel Dashboard → проект → можно подключить репозиторий или спросить помощь с первой настройкой Git.

## 5. Проверка, что всё работает

### Шаг 5.1. Health API

Откройте в браузере:

```
https://www.bess-motors.com/api/health
```

Должно быть `"ok": true` и зелёные пункты:

- `supabase` — ok  
- `admin_env` — ok  
- `jwt_secret` — ok  

### Шаг 5.2. Вход в CRM

Повторите [раздел 2](#2-вход-в-crm-продакшен) в **инкогнито**.

### Шаг 5.3. В CRM → Настройки

Откройте вкладку **Настройки** (Settings) в CRM.  
Блок **«Integracje serwera»** / интеграции — основные пункты должны быть **зелёные**.

### Шаг 5.4. Данные в облаке

1. Откройте заказ-наряд или список клиентов.  
2. Подождите 10–20 секунд после входа (синхронизация).  
3. Если пусто — проверьте Supabase (таблица CRM) или интернет.

## 6. Работа в CRM каждый день

### Быстрый старт

1. https://www.bess-motors.com/cabinet?crm=1 → войти (если сессия сброшена).  
2. https://www.bess-motors.com/crm — обзор дня.  
3. **Горячие заказы** — новые звонки и записи.  
4. **Заказ-наряды** — создать / изменить / PDF / подпись.

### Заказ-наряд

| Действие | Где |
|----------|-----|
| Новый заказ | CRM → Заказ-наряды → «+» |
| Подпись клиента | Ссылка на подпись, SMS, WhatsApp (ручная кнопка WA) |
| Сообщение клиенту | В карточке заказа: **TG** / **WA↑** / **WA** |
| Статус «Готов» | Поле статуса → сохранить (клиенту уйдёт уведомление, если подключён Telegram) |

### Клиент vs админ

- **Клиенты** заходят на `/cabinet` с **госномером** авто.  
- **Вы (админ)** — телефон + **пароль** с Vercel.

## 7. Сайт на своём компьютере (локально)

Если нужно проверить без интернета:

### Шаг 7.1. Установить Node.js

https://nodejs.org — версия **20 LTS** или **22**.

### Шаг 7.2. Файл `.env.local`

В папке `bess-motors` создайте файл `.env.local` (не выкладывать в Git!):

```env
ADMIN_PHONE=+48888838688
ADMIN_PASSWORD=ваш_пароль
JWT_SECRET=dev-local-jwt-only-not-for-production
NEXT_PUBLIC_SITE_URL=http://localhost:3000
SUPABASE_URL=...
SUPABASE_SERVICE_ROLE_KEY=...
```

(Скопируйте Supabase с Vercel.)

### Шаг 7.3. Запуск

```powershell
cd c:\Users\Asus\Desktop\website\bess-motors
npm install
npm run dev
```

Браузер: http://localhost:3000/cabinet?crm=1

Остановить: `Ctrl + C`.

### Если PowerShell пишет про «сценарии»

```powershell
npm.cmd run dev
```

## 8. Если не входит / «криво» работает

### «Неверный телефон или пароль администратора»

| Проверка | Действие |
|----------|----------|
| Второе поле | Это **пароль**, не номер машины |
| Vercel | `ADMIN_PHONE` и `ADMIN_PASSWORD` совпадают с тем, что вводите |
| Пробелы | Пересоздайте пароль в Vercel без пробелов |
| Deploy | После смены переменных — **Redeploy** |
| Код на сервере | Нужен push с исправлением входа (раздел 4) |

### Входит и сразу выбрасывает на вход

| Причина | Решение |
|---------|---------|
| Старый токен в браузере | Инкогнито |
| Нет `JWT_SECRET` на Vercel | Добавить и Redeploy |
| Старая версия сайта без fix | Deploy из раздела 4 |

### «Облако CRM недоступно»

| Причина | Решение |
|---------|---------|
| Supabase | Проверить `SUPABASE_URL` и `SUPABASE_SERVICE_ROLE_KEY` |
| Проект Supabase | Не на паузе, есть таблица CRM |

### CRM пустой (нет заказов)

| Причина | Решение |
|---------|---------|
| Синхронизация | Подождать 20 сек, обновить страницу |
| Другой браузер | Данные в облаке общие; локальный кэш может быть старым |
| Supabase пустой | Данные только на старом ПК — нужен перенос |

### Сайт «криво» на телефоне

CRM удобнее с **компьютера**. На телефоне — звонки, WhatsApp; тяжёлые формы — с ПК.

### Тест API входа (для техподдержки)

После deploy, в консоли браузера (F12) на странице входа при ошибке смотрите вкладку **Network** → запрос `staff-login` → код 401 или 200.
